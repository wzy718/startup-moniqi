# 场景查看说明

## 问题：场景编辑器中看不到节点

### 原因分析
1. **Canvas 的 UITransform 显示大小为 100x100**，在编辑器中可能显示太小
2. 虽然 `alignCanvasWithScreen` 为 true，但编辑器中的显示可能不准确
3. 节点实际存在，但可能在编辑器的可视范围外

### 解决方法

#### 方法 0：重新加载场景（最重要！）
**Cocos Creator 中没有 "Open Scene" 菜单项，正确的方法是：**

1. **在 Assets 面板中双击场景文件**（推荐）
   - 在左下角的 `Assets` 面板中
   - 展开 `assets` → `scenes`
   - **双击 `Main.scene` 文件**
   - 这会强制重新加载场景

2. **或者右键点击场景文件**
   - 在 `Assets` 面板中，右键点击 `Main.scene`
   - 选择 `打开场景` 或 `Open Scene`

3. **如果还是看不到节点，尝试：**
   - 关闭 Cocos Creator
   - 删除 `cocos/temp/` 和 `cocos/library/` 文件夹（缓存文件夹）
   - 重新打开 Cocos Creator 和项目
   - 再次双击 `Main.scene` 打开场景

#### 方法 1：在 Hierarchy 面板中查看节点（推荐）
1. 打开左侧的 **Hierarchy** 面板
2. 展开 `Canvas` → `UIRoot` → `MainPanel`
3. 你应该能看到所有子节点：
   - TopBar
   - StatusArea
   - WeeklyInfo
   - ShopArea
   - EventArea
   - BottomNav

#### 方法 2：调整场景视图
1. 在场景编辑器中，使用鼠标滚轮**放大视图**
2. 或者按 `F` 键**聚焦到选中的节点**
3. 在 Hierarchy 中选中 `Canvas`，然后按 `F` 键

#### 方法 3：检查项目设置
1. 菜单：`项目` → `项目设置`（或 `Project` → `Project Settings`）
2. 检查 `设计分辨率` 是否为 `750 x 1334`
3. 检查 `适配屏幕宽度` 是否勾选

#### 方法 4：直接预览运行
1. 点击编辑器顶部的 **预览** 按钮（或按 `Ctrl+P` / `Cmd+P`）
2. 在浏览器预览中应该能看到 UI 界面
3. 如果预览正常，说明场景配置正确，只是编辑器显示问题

### 验证节点是否存在
在 Hierarchy 面板中，你应该能看到以下结构：
```
Canvas
├── Background (y=3000, 隐藏)
├── TitleLabel (y=3000, 隐藏)
├── StatusLabel (y=3000, 隐藏)
├── StartButton (y=3000, 隐藏)
├── UIRoot
│   ├── MainPanel (active: true)
│   │   ├── TopBar
│   │   ├── StatusArea
│   │   ├── WeeklyInfo
│   │   ├── ShopArea
│   │   ├── EventArea
│   │   └── BottomNav
│   ├── EventPanel (active: false)
│   ├── ShopDetailPanel (active: false)
│   └── SettlementPanel (active: false)
└── Main Camera
```

### 如果 Hierarchy 中也没有节点
1. 检查场景文件是否正确加载
2. 尝试重新打开场景：`文件` → `打开场景` → 选择 `assets/scenes/Main.scene`
3. 如果还是不行，可能是场景文件损坏，需要从 Git 恢复
